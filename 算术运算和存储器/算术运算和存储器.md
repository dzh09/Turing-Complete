### 成对的麻烦

---

#### 关卡要求：实现当有两个及以上的高电平的时候，输出高电平。剩余情况实现输出低电平。

![](算术运算和存储器.assets/屏幕截图 2026-01-25 203443.png)

**这道题就不能在观察真值表了，因为输入信号有四个，情况太多了。所以可以用公式化简。[^1]** 

[^1]:前面的关卡都可以用公式化简，好处是通过化简计算一定能够得到结果，但是计算不方便太过于无脑，丧失这个游戏本身的乐趣了

**这里把四个信号定义为ABCD，然后可以将所有的高电平输出的情况罗列出来**

| 输入情况 | 所有可能          |
| -------- | ----------------- |
| 2个1     | AB AC AD BC BD CD |
| 3个1     | ABC ABD ACD BCD   |
| 4个1     | ABCD              |

**AB+AC+AD+BC+BD+CD+ABC+ACD+BCD+ABCD**
**=AB(1+C+CD)+AC(1+D)+AD+BC(1+D)+BD+CD**	
**=AB+AC+AD+BC+BD+CD**
**=A(B+C+D)+B(C+D)+CD**



### 奇数个信号

-----

#### 关卡说明：当输入信号有奇数个为高电平输出为高电平，并且不能使用超过三个元件

#### 提示：两路之间的奇数判断只需要用到1个元件

![](算术运算和存储器.assets/屏幕截图 2026-01-25 214335.png)

**因为两路之间的奇数判断只用一个元件，所以我们需要找到一个符合奇数输出高电平的元件**

**异或门 同则取零异则取一 也就是说只有一个输入为高电平的时候才输出高电平 符合**

**那这一题就可以看作需要一个四位异或门。**

**由前面的三路与门和三路或门 我们可以大胆猜测这里的四路异或门是由三个异或门组成**

**我们可以验证：当信号为奇数个的时候，只能为1或3。上面为0下面为1或者上面为1下面为2**

**我们可以发现无论哪种情况一定是一路为奇数个高电平 一路为偶数个高电平**

**所以两个异或门的输出一定是不同的电平 那为了确保最后输出是1 就必须放一个异或门 异则取1**



**同理，如果把前面两个异或门换成同或门 仍然可以**

**因为奇数个信号1和3不论那种总是能 一路偶数个高电平 一路奇数个高电平**

**一路相同的信号 一路不同的信号 所以即便执行相反的逻辑 得到的两个输出只是上下调换顺序而已**

**最后再用一个异或门判断就好**

### 信号计数

----

#### 关卡说明：这是一个二进制计数器，输出的三个引脚分别代表二进制的1、2、4。通过计数输入信号中高电平的数量，在输出引脚表示

![](算术运算和存储器.assets/屏幕截图 2026-02-08 104630.png)

- 二进制的最低位

  观察发现只有当信号中有奇数个高电平时，这一位输出高电平

  这个和前面关卡“奇数个信号”的逻辑一样

- 二进制的第二位

  观察发现当信号中有两个以上且不全为高电平时，这一位输出高电平

  这个和前面关卡“成对的麻烦”基本相似，只需要在此基础上，保证全为高电平的时候这一位输出低电平就好

  | 信号计数             | 0    | 1    | 2    | 3    | 4    |
  | -------------------- | ---- | ---- | ---- | ---- | ---- |
  | 第二位（成对的麻烦） | 0    | 0    | 1    | 1    | 1    |
  | 最高位               | 0    | 0    | 0    | 0    | 1    |
  | 第二位（目标输出）   | 0    | 0    | 1    | 1    | 0    |

  观察真值表发现，满足异或门的逻辑，在第二位和第三位之间加一个异或门

- 二进制的最高位

  观察发现只有所有信号都是高电平的时候，这一位输出高电平

  这个是&与的逻辑，所以就用一个三路与门和一个与门

### 全加器

----

#### 关卡说明：三位二进制加法，进位输出在CAR引脚，低位输出在SUM引脚

![](算术运算和存储器.assets/屏幕截图 2026-02-08 104740.png)

**全加器就是两个半加器的嵌套**

- SUM

  异或门的逻辑，三位异或

- CAR

  半加器是与门的逻辑，全加器就是两个半加器的与门得到的结果相加（+就是逻辑或）

![](算术运算和存储器.assets/屏幕截图 2026-02-08 104752.png)
